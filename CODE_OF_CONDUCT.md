# Code of Conduct

## Overview

This repository contains proprietary source code owned by Indigo Labs LLC. While the
source code is publicly viewable for review and educational purposes, this is not a
traditional open-source project with community governance.

## Scope and Intent

**This repository is public for:**
- Transparency and code quality demonstration
- Educational reference
- Security review by authorized parties
- Collaboration with explicitly approved partners

**This repository is NOT:**
- An open-source project accepting general community contributions
- A community-governed project
- Available for forking and derivative works without written permission

## Interaction Standards

All individuals interacting with this repository must:

1. **Respect Intellectual Property**
   - Acknowledge this is proprietary software
   - Do not create unauthorized derivative works
   - Do not use code in competitive products
   - Follow the terms of the LICENSE file

2. **Professional Conduct**
   - Be respectful in all interactions
   - Use professional language in issues and discussions
   - Focus on constructive feedback
   - Respect that final decisions rest with Indigo Labs LLC

3. **Security and Privacy**
   - Do not share sensitive information in public issues
   - Report security vulnerabilities privately (see SECURITY.md)
   - Do not attempt to access restricted resources
   - Respect healthcare data privacy requirements

4. **Authorized Contributions Only**
   - Contributions require explicit written permission from Indigo Labs LLC
   - Unsolicited pull requests will be closed
   - See CONTRIBUTING.md for the contribution process
   - All accepted contributions become property of Indigo Labs LLC

## Prohibited Behavior

The following will result in blocking and potential legal action:

❌ Creating unauthorized forks or derivative works
❌ Using code in competitive healthcare products
❌ Removing or altering copyright notices
❌ Harassment, threats, or abusive behavior
❌ Spam, promotional content, or off-topic discussions
❌ Attempting to circumvent access controls
❌ Reverse engineering proprietary elements
❌ Misrepresenting affiliation with Indigo Labs LLC or Lighthouse HLTH

## Issue and Discussion Guidelines

When creating issues:

✅ **Allowed:**
- Bug reports with reproducible steps
- Questions about using the library (if you're an authorized user)
- Security vulnerability reports (via private channel only)
- Documentation clarifications

❌ **Not Allowed:**
- Feature requests (unless you're an authorized partner)
- General support questions (use official support channels)
- Philosophical debates about licensing
- Attempts to negotiate license terms via issues

## Enforcement

Indigo Labs LLC reserves the right to:
- Close issues or discussions without explanation
- Block users who violate these guidelines
- Remove content that violates the LICENSE terms
- Take legal action for intellectual property violations
- Modify these guidelines at any time
